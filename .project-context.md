# 🗺️ Project Context: QuanLyDoanKham

File này được thiết kế để giúp AI hiểu ngay cấu trúc và mục tiêu dự án mà không cần quét lại toàn bộ mã nguồn, giúp **tiết kiệm Token** và **thời gian**.

## 🚀 Tổng quan dự án
- **Tên dự án:** Quản Lý Đoàn Khám (QuanLyDoanKham)
- **Mục tiêu:** Quản lý quy trình khám sức khỏe cho các đoàn (công ty, trường học), từ khâu tiếp nhận, khám lâm sàng đến trả kết quả và xuất báo cáo.
- **Tính năng chính:** Quét mã QR, quản lý danh sách bệnh nhân, nhập kết quả khám, tạo file PDF/Excel.

## 🛠️ Nền tảng công nghệ (Tech Stack)
### Frontend (`/QuanLyDoanKham.Web`)
- **Framework:** Vue 3 (Composition API) + Vite
- **State Management:** Pinia
- **Styling:** Tailwind CSS
- **Thư viện quan trọng:** 
  - `html5-qrcode`: Quét mã QR trên trình duyệt/mobile.
  - `@microsoft/signalr`: Giao tiếp thời gian thực với Backend.
  - `lucide-vue-next`: Icon hệ thống.

### Backend (`/QuanLyDoanKham.API`)
- **Framework:** ASP.NET Core (.NET 10.0)
- **Database:** SQL Server (Entity Framework Core)
- **Thư viện quan trọng:**
  - `QuestPDF`: Tạo báo cáo PDF chuyên nghiệp.
  - `ClosedXML`: Xử lý file Excel.
  - `SignalR`: Đẩy thông báo thời gian thực.

## 📂 Cấu trúc thư mục quan trọng
- `/docs`: Chứa tài liệu quy trình, sơ đồ BPMN và hướng dẫn sử dụng.
- `/QuanLyDoanKham.API/Controllers`: Chứa các API endpoints.
- `/QuanLyDoanKham.Web/src/views`: Chứa giao diện các trang chức năng.

## 🎯 Quy trình nghiệp vụ (Business Logic)
1. Tiếp nhận đoàn khám -> Tạo mã QR cho từng bệnh nhân.
2. Bệnh nhân đi qua các phòng khám -> Bác sĩ quét QR và nhập kết quả.
3. Hệ thống tổng hợp kết quả -> Xuất báo cáo tổng hợp (Excel/PDF).
